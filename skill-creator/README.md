# Skill Creator - 使用说明

用于创建新agent skill的元技能。帮你把一个想法或已有实践提炼为标准化、可复用的skill。

## 快速开始

### 我有一个想法，想让agent帮我探索实现

```
我想开发一个skill：[描述你想要的能力]。
先进入探索阶段，你来自主探索实现方式。
```

agent会自主尝试、记录关键步骤，探索完成后提炼为skill。

### 我想一步步指导agent实现

```
我想开发一个skill：[描述你想要的能力]。
进入探索阶段，我来逐步指挥你操作。
第一步：[你的指令]
```

你逐步给出指令，agent执行反馈，打通后说"进入提炼阶段"即可。

### 我已经知道怎么做，直接生成skill

```
帮我做一个skill：[描述目标和关键步骤]
```

agent会直接按模板生成，无需探索阶段。

### 我有现成文档，想转成skill

```
把这个文档提炼成skill：
[粘贴文档内容，或提供文件路径]
```

## Prompt技巧

- **描述目标而非实现**：说"我想要一个能自动格式化代码的skill"比"帮我写一个调用prettier的脚本"更好
- **说明使用场景**：帮助agent判断skill类型和设计粒度
- **提供约束条件**：如"只支持Python项目"、"不依赖外部服务"等

## 产出物

**单体 skill** 生成到 `skills/<skill-name>/`：

| 文件 | 说明 |
|------|------|
| `SKILL.md` | 核心文档，agent读取并执行 |
| `README.md` | 使用说明，面向用户 |
| `config.example.yaml` | 配置模板（如需要） |
| `scripts/*.sh` | 工具脚本（如需要） |

**Skill suite（skills 集合）** 生成到 `skills/<suite-name>/`，包含入口 `SKILL.md`（路由表）、`_lib/`（共享脚本）和若干子模块目录，每个子模块结构同单体 skill。

## 注意事项

- 一个skill只做一件事，过大应拆分
- 探索阶段不急于总结，充分验证后再提炼
- 提炼后建议在新会话中验证skill是否能被agent正确执行
