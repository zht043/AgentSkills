# ssh-dev-suite

SSH远程开发套件，包含连接管理、命令执行、文件传输、项目部署、端口隧道、远程调试。

## 模块

| 模块 | 功能 |
|------|------|
| connect | 连接管理、命令执行、文件传输、后台任务 |
| deploy | 项目部署与回滚 |
| tunnel | 端口转发与代理 |
| debug | 结构化远程问题排查 |
| long-task | 长耗时任务管理、checkpoint记忆、跨会话恢复 |

## 快速开始

### 配置连接
```
帮我配置SSH连接到我的开发服务器
```
agent会引导填写服务器信息，自动生成config.yaml。

### 常用Prompt

**连接与执行：**
- `测试一下 dev-server 的连接`
- `在 dev-server 上执行 ls -la`
- `帮我在远程启动一个后台训练任务`

**部署：**
- `帮我把当前项目部署到dev-server`
- `回滚上一次部署`
- `帮我配置部署流程`

**隧道：**
- `帮我把远程服务器的5432端口映射到本地`
- `服务器没有外网，帮我设置代理`

**调试：**
- `服务器上的 my-app 服务挂了，帮我排查`
- `检查一下 dev-server 的资源使用情况`

**长耗时任务：**
- `帮我在远程启动模型训练，预计2小时`

- `把远程任务日志流式输出到本地文件`

## 环境变量设置

密码通过环境变量管理，不写入配置文件。

**Windows：**
```cmd
setx SSH_DEV_PASSWORD "你的密码"
```

**macOS / Linux：**
```bash
echo 'export SSH_DEV_PASSWORD="你的密码"' >> ~/.bashrc
source ~/.bashrc
```

## 认证方式

| 方式 | 说明 |
|------|------|
| SSH密钥（推荐） | config.yaml设置`identity_file`，最安全便捷 |
| 密码 | config.yaml设置`env_password`为环境变量名，依赖sshpass |

优先级：SSH密钥 > 环境变量密码 > 交互式提示

## 注意事项

- 配置由agent引导生成，无需手动编辑
- config.yaml含敏感信息不入库
- 会话复用：首次连接后SSH保持连接（默认600秒）
- 容器支持：配置container字段后命令自动在容器内执行
- 跳板机：配置jump_host后自动中转
